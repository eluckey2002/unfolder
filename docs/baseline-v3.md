# v3 quality baseline

This document is the **frozen "v3 starts here" snapshot** — the
quality-metric capture taken at the start of v3 (session 0021). It
is the comparison point against which v3 improvements (optimized
recut, smart tab placement, packing improvements) are measured.

`docs/baseline-pipeline.md` is the *live* baseline, regenerated by
`pnpm baseline` and tracked as the current state of the pipeline.
This file is a one-time copy of it, captured at v3's start and
never regenerated. CI guards both against unintended change; the
`baseline-change` label is the escape hatch for an intentional
regeneration.

The five metrics, each tied to a v3 improvement vector:

- **piece count** — fragmentation (targeted by optimized recut)
- **cut length (mm)** — assembly effort (smarter cuts → less cutting)
- **tabs** — assembly effort (smart tab placement → fewer/better tabs)
- **paper efficiency** — paper waste (packing + scale)
- **pages** — output size
