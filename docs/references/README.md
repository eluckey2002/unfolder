# Reference Implementations

This directory holds writeups of other papercraft/unfolding implementations we've studied.

One markdown file per project. Coverage is loose, but each writeup should address:

- **What it is** — project name, language, rough scope
- **Algorithm/approach** — how it solves the unfolding problem
- **What we learned** — anything directly relevant to our work
- **What we chose not to copy** — explicit non-decisions, and why

Examples of projects worth documenting: paperfoldmodels, PolyZamboni, and similar tools found during research.
