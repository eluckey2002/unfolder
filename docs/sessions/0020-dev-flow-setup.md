# Session 0020 — Development flow setup

## What was attempted

Stand up the v3 development flow: a PR-based merge path with CI as
the safeguard layer. The session creates ADR 0006, the PR template,
the CI baseline guard, the descriptive branch-naming convention, the
prompt-preflight checklist, and the doc-coherence updates that bring
`CLAUDE.md`, `project-state.md`, and `roadmap.md` in line with the
new flow.

This is a docs-and-infra session — no `src/` code is touched. It
runs in a worktree and ends with a session log, but it runs under
the *old* flow (worktree → fast-forward to `main`): it is building
the PR flow, so it cannot run under it. Session 0021 is the first to
run PR-style.

## What shipped

- **`docs/decisions/0006-pr-based-development-flow.md`** — the
  decision record. v2 went public with CI but the workflow only ran
  on push to `main`, gating nothing. The recurring v1/v2 failures —
  outside-chat drift, a prompt edited after handoff, an unwritten
  branch-naming rule, the CLAUDE.md corpus table drifted two corpora
  out of sync — share one shape: discipline that decayed because
  nothing structural held it. v3 makes the discipline load-bearing
  structure: PR-based merges, branch protection, descriptive branch
  names, a PR template that *is* the structural home of the handoff
  block, a CI baseline guard, and a prompt-preflight checklist. ADR
  0006 also defines a doc-ownership map (ADR = *why*, project-state
  = canonical *what*, CLAUDE.md = operative subset) so the same fact
  stops drifting across three documents. Linter, diff-aware nudges,
  size-threshold warning, and process-lint are explicitly held as
  fast-follow, not built here.

- **`.github/pull_request_template.md`** — the structural form of
  the session handoff block. Sections: Summary; Verification
  checklist (type-check, test:run with reported total, build,
  baseline-unchanged-or-label); Spec adherence & scope; Decisions
  tagged by autonomy tier (`[flowed-silently]` /
  `[surfaced-and-proceeded]` / `[needs-Evan]`); Concerns,
  uncertainties, questions; Doc coherence checkline; Queue/roadmap
  deltas; Links; paste-ready squash commit message; Merge-readiness
  checklist. Mirrors the in-repo session log, which stays canonical.

- **`.github/workflows/ci.yml`** — adds a `Baseline guard` step to
  the existing `verify` job, after `pnpm build`. The step runs
  `pnpm baseline`, then fails the job if `docs/baseline-pipeline.md`
  is no longer byte-identical, *unless* the PR carries a label named
  `baseline-change`. The diff uses `git diff --exit-code -I '^Generated by'`
  to ignore the volatile date-stamp header that
  `scripts/baseline-pipeline.ts` rewrites on every run. Label
  presence is read from the standard
  `github.event.pull_request.labels.*.name` array via the
  `contains()` expression and exposed as an env var
  `BASELINE_CHANGE_ALLOWED`. The existing `push: main` trigger stays
  — it becomes a post-merge backstop; a failure there means
  something bypassed the PR flow.

- **`docs/strategist-protocol.md`** — appended a `Prompt preflight
  checklist` section as the final section of the doc. Nine items, each
  tied to a known v1/v2 failure class (no call signatures from memory,
  no predicted test counts, formal syntax is exact, verification steps
  named, files and merge strategy named, new tooling probed, prompt
  self-contained, work type and branch name declared, prompt frozen
  after handoff). A subagent runs the checklist; the strategist
  triages.

- **`CLAUDE.md`** — three coordinated changes:
  - Section 1 rewritten to be a triggers-only table that points to
    `project_corpora.md` as the canonical inventory (the table here
    no longer carries coverage counts that drift); the subagent

  - Section 3 loses one bullet — the "decisions-corpus" build

  - A new `## 6. Development workflow (v3+)` section gives the

    PR, fill the PR template) and points to ADR 0006 for reasoning,
    `project-state.md` for the working agreement. A new bullet in
    Section 5 adds `git worktree prune` at session start.

- **`docs/project-state.md`** — two edits:
  - "Session and commit mechanics" gains a new bullet (right after
    the existing worktree-by-default bullet) committing numbered
    sessions and spikes to land via PR per ADR 0006.
  - The "Where to look" ADR list adds ADR 0006.

- **`docs/roadmap.md`** — two edits:

## Concerns, uncertainties, questions

- **The CI baseline guard is untested until session 0021 opens its
  first PR.** The YAML parses, the `contains(github.event.pull_request.labels.*.name, …)`
  expression is standard, and the existing `verify` job structure is
  intact — but the new step has not been exercised against a live PR
  yet. Confidence: high on syntax (current GitHub Actions
  convention), medium on label semantics (the `baseline-change`
  label name has to be created on the repo before it can be applied
  to a PR — worth surfacing to Evan as part of the GitHub-side
  setup).
- **Pre-merge amendment freedom under the new flow.** Worktree
  commits are still drafts pre-merge, but the *path* to `main` is
  now PR → squash-merge, not fast-forward. Amending a worktree
  commit and force-pushing the branch still works pre-merge; the
  squash-merge collapses to one clean commit on `main` regardless.
  This should be fine in practice but is worth confirming with the
  first 0021 PR.
- **No follow-up queue entry was added for the fast-follow items.**
  ADR 0006 names them in prose; if Evan wants them tracked in
  `docs/queue.md` as well, that's a quick add.
