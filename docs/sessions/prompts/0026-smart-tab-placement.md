## Goal

Replace `src/core/tabs.ts`'s "lower mesh-face index gets the tab" rule with a score-driven choice between the two candidate sides of each cut edge. Score by edge length minus a hard penalty when the candidate tab polygon would overlap the originating piece's own interior. Net effect is conservative — score-driven placement matches the existing lower-face-index rule wherever both candidate sides are clean (the common case across the corpus); only diverges where one side's candidate tab would overlap another face in the same piece. Reshapes placement primarily on `croissant.obj`, `deer.obj`, `ginger-bread.obj` — the corpus's multi-piece (and so most topologically complex) models.

## Specs

- **`scoreTabPlacement`** — exported pure scorer over a placement signal. The signal carries (a) an edge length in mm and (b) a boolean flag for whether the candidate tab polygon overlaps the originating piece's interior. The score combines a positive edge-length term with a hard negative penalty when the overlap flag is true. The constraint: the penalty must dominate any reasonable edge-length term so an overlap-free candidate always wins regardless of edge length. Seed values: `W_LENGTH=1.0`, `W_OVERLAP=1000`. The constraint sentence is the spec; the numbers are seed values that can shift during implementation (record any change in the Task 26.6 decisions-log entry).
- **`tabOverlapsOwnPieceInterior`** — exported pure predicate. Takes a candidate tab polygon and a piece's faces (plus the originating face's index so the predicate can skip it); returns true iff the polygon intersects any other face's positions polygon with non-zero area. Uses the `polygonClipping.intersection` pattern from `src/core/overlap.ts`; mirrors the try/catch defense from that file (treat thrown exceptions as non-overlap, same numerical reasoning).
- **`buildRenderablePieces`** becomes two-pass: pre-pass decides per-cut globally; main pass looks up the winning side. Tie-break to `adj.faceA` (preserves existing fixture expectations on the symmetric two-piece case).
- **`scripts/baseline-pipeline.ts`** gains a `tab overlap (own)` column. After Task 26.3 lands, expect 0 on every row across the 11-model corpus — if any row > 0 the algorithm has a bug, go back to Task 26.3 Step 3.
