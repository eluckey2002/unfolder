### ADR 0008 draft — Region-aware pipeline contract (v3.5)

```markdown
# ADR 0008 — Region-aware pipeline contract (v3.5)

## Context

v3.5 introduces the ability to re-unfold a connected region of a mesh's
faces without perturbing the rest of the current layout. This capability
backs:

- v4.1 Subset B's edge-toggle and undo/redo — a re-unfold is the atomic
  operation undo replays via `appliedFixes` traversal.
- v4.3's fix-suggestion engine "re-unfold region with alt weights" fix
  family — the engine's region-fix backend.

The 0031 spike (`docs/spikes/2026-05-17-v3.5-region-reunfold/`) produced
the algorithm specification and a verdict of NEEDS-REVISION pending
four v4 spec edits (three to § 7, one to § 4; applied in 0032 Task 1) and
deer-scale validation (0032 Task 4). With those, the algorithm is
structurally implementable, deterministic, and cost-bounded.

The v3 pipeline (ADR 0007 greedy cut-removal as default) consumes no
RNG; v3.5 preserves this property — determinism is over the inputs.

**Notation:** `u32` in this ADR is a documentation convention indicating
"32-bit unsigned integer range" on a TypeScript `number`. No nominal
type is introduced; the field at `src/app/store.ts` `AppliedFix.seed`
remains `number` with a JSDoc `/* u32 */` annotation.

## Decision

Adopt the region re-unfold algorithm specified in
`docs/spikes/2026-05-17-v3.5-region-reunfold/algorithm.md` (with the
0032 spec edits applied) as the v3.5 contract. Concretely:

1. **New exports in `src/core/`:**
   - `buildRegionAdjacency(mesh, region) → { dual, openBoundary }` —
     region-aware variant of `buildAdjacency` that treats edges shared
     with non-region faces as open boundaries instead of non-manifold
     violations.
   - `reUnfoldRegion(inputs) → RegionReUnfoldResult` — the six-step
     engine returning either `OK` with `{ spliced, replay, diagnostics }`
     or `FAIL` with a typed `reason`.

2. **Determinism contract.** Each accepted re-unfold returns a
   `replay: { seed: number /* u32 */; edgeOrderHash: string }` pair.
   `edgeOrderHash` is the SHA-256-prefix (first 128 bits hex) of the
   region's dual-adjacency iteration order — canonically sorted by 3D
   edge length descending with explicit tie-breaks per
   `algorithm.md` § 3. `seed` is reserved for future use; v3.5 consumes
   no RNG and the replay `seed` is always `0`. Replay against a pinned
   `(seed, edgeOrderHash)` produces a byte-identical `spliced` layout.
   The v4.3 determinism property test exercises this.

3. **Scope boundary: boundary edges always remain cuts post-re-unfold.**
   Re-unfold cannot fold the region/pinned boundary into a pinned
   non-region face — that would require moving the pinned face's 2D
   position, violating the pinned-outside contract. The user's recovery
   path for cross-region restructuring is the v3.5 fallback (manual
   region-pin + full-pipeline-re-run with loading spinner) per v4 spec § 3.

4. **Page-aware splice.** The splice step removes region faces from any
   affected pieces (splitting if a piece straddled the region), adds the
   new local pieces to the affected pages, and repaginates ONLY the
   affected pages. Non-region pieces on non-affected pages retain their
   exact `placed.transform` values bit-exactly.

5. **Failure handling.** `RegionReUnfoldResult.FAIL` carries a typed
   `reason` enum (`obstacle-unresolvable`, `region-overlap`,
   `post-splice-overlap`, `region-disconnected`,
   `anchor-pinning-precondition-violated`). v4.3's fix-suggestion
   PREDICT → SCORE pipeline rejects FAIL fixes with `SCORE = −∞`; the
   user-facing recovery is "manual edit" (v4.3 edge-click) or the v3.5
   fallback path.

## Status

Accepted (session 0032).

## Consequences

**Positive:**

- v4.1 Subset B (edge-toggle, undo/redo) has a deterministic substrate
  per v4 design spec § 4's determinism property.
- v4.3's region re-unfold fix family ships against a documented
  contract.
- v4.0 store's `appliedFixes` shape (session 0030) is honored; no
  migration when v4.1 Subset B implements the traversal.
- Coherent with ADR 0007 (cut-removal as v3 default): the region
  pipeline calls the same recut algorithm the production pipeline does.

**Negative / accepted tradeoffs:**

- `seed` carries forward as a reserved field with no consumer; mildly
  awkward in the type shape until v4.4+ introduces an RNG path.
  Accepted to preserve the v4.0 store-shape lock.
- The rotation/translation grid (12 × 15°, ±8 × median-edge per
  `algorithm.md` § 2 Step 5) is empirically chosen; if a future workload
  finds it insufficient, parameter values tune within the algorithm's
  mechanism, not the contract.
- v3.5 scope boundary forbids cross-region folding — the fallback path
  is the user's escape hatch but is multi-second slow. Accepted to keep
  the v3.5 contract bounded.
- No texture / UV passthrough on re-unfold — v3.5 inherits v3's Kd-only
  color passthrough; textures land in v5 per README phase plan.
```
