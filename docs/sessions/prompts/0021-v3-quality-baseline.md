# Session 0021 — v3 quality baseline

## Goal

Establish v3's failure baseline. Define the quality metric set,
instrument `scripts/baseline-pipeline.ts` to compute it, and capture
`docs/baseline-v3.md` as the frozen "v3 starts here" snapshot — the
north star against which v3's optimized-cut, smart-tab, and packing
work each measure themselves.

This is also **the first session run under the PR flow** ADR 0006
stood up. The worktree → PR → CI → squash-merge path applies. The
prompt-preflight checklist has already been run on this prompt.

## Context

v2 closed with overlap-free, buildable papercraft on the 11-model
corpus. But the v2 baseline measures only "did the pipeline complete"
and per-model piece + page counts — it does not measure *quality*.
v3's improvement vectors (optimized recut, smart tab placement,
packing) each need a number to move. This session adds the metric
set that gives them targets.

Per the v3 plan in `docs/roadmap.md`, the metric set is five
measures, each tied to an improvement vector: piece count
(fragmentation), total cut length (assembly effort), tab count
(assembly effort), paper-area efficiency (paper waste), page count
(output size). Foldability is deferred — no clean measurable
definition yet.

2. **Extend `scripts/baseline-pipeline.ts`** with three new metric

   - **`cut length (mm)`** — total length of all cut-classified
     edges across all renderable pieces, after paginate's uniform
     scale. The existing script already produces renderable pieces
     in the pipeline path; extend that path to walk each piece's
     edges, summing the 2D mm-length of every edge classified as a
     cut. Report rounded to one decimal place. (Under rigid
     unfolding, 3D and 2D-post-scale edge lengths are equal; pick
     whichever is cleaner to read off the data the script already
     has in hand.)
   - **`tabs`** — total tab count across all pieces. Under the
     lower-face-index rule (see `src/core/tabs.ts`) each cut edge
     produces one tab, so the count equals the size of the cut set
     `recut` returns. Locate the exact field name in the source
     before relying on it.
   - **`paper efficiency`** — used face area divided by total
     printable page area, as a percentage to one decimal place. Sum
     2D face-triangle areas (in mm², post-paginate scale) across
     all pieces on all pages; divide by the page count times the
     printable area per page. The `LETTER` `PageSpec` (already
     imported into the script from `src/core/paginate.ts`) carries
     width, height, and margins in mm — read the actual field names
     from that source, do not assume them.

## Appendix A — `docs/baseline-v3.md` preface (verbatim)

```markdown
# v3 quality baseline

This document is the **frozen "v3 starts here" snapshot** — the
quality-metric capture taken at the start of v3 (session 0021). It
is the comparison point against which v3 improvements (optimized
recut, smart tab placement, packing improvements) are measured.

`docs/baseline-pipeline.md` is the *live* baseline, regenerated by
`pnpm baseline` and tracked as the current state of the pipeline.
This file is a one-time copy of it, captured at v3's start and
never regenerated. CI guards both against unintended change; the
`baseline-change` label is the escape hatch for an intentional
regeneration.

The five metrics, each tied to a v3 improvement vector:

- **piece count** — fragmentation (targeted by optimized recut)
- **cut length (mm)** — assembly effort (smarter cuts → less cutting)
- **tabs** — assembly effort (smart tab placement → fewer/better tabs)
- **paper efficiency** — paper waste (packing + scale)
- **pages** — output size
```
