# Session 0031 — v3.5 region re-unfold spike

Date: 2026-05-17. Branch: `spike/0031-region-reunfold`.

The first session of the v3.5 incremental-pipeline phase. A
docs-only spike that designs the region re-unfold algorithm,
worked-traces it on a synthetic minimal mesh, and surfaces every
underspecification in v4 spec § 7's pseudocode block (lines 302–345).
Output is two markdown deliverables; no production code changes.
The companion paired session is 0032 (promote-to-core + ADR).

## What was attempted

- Read v4 spec § 7 in full and enumerated every underspecification
  in the pseudocode block. Includes both the prompt's named
  candidates (six) and two surfaces only by reading the spec
  against the actual v3 source code (the spec's
  `buildSpanningTree + flatten + recut` pseudocode is two ADRs
  out of date; the spec's "seed: u32" field has no consumer in
  v3 today).
- Drafted the algorithm spec at
  `docs/spikes/2026-05-17-v3.5-region-reunfold/algorithm.md` —
  input/output contract, preconditions, six-step procedure with
  per-step rationale and edge cases, failure modes, determinism
  contract with a precise `edgeOrderHash` definition, API sketch,
  worked-example trace on `octahedron.stl`.
- Worked-example trace: 2-of-8-face region on `octahedron.stl`
  hand-traced through every algorithm step (boundary partition,
  anchor selection, region adjacency, region pipeline + anchor
  transform, obstacle resolution, splice + verify). Hand-traced
  because the spike worktree had no `node_modules` and the prompt
  forbade silent `pnpm install` (the harness correctly blocked an
  install attempt). The deer-scale demonstration is reserved for
  session 0032's first task.
- Wrote findings at
  `docs/spikes/2026-05-17-v3.5-region-reunfold/findings.md` with
  the three required sections: verdict, evidence, next step.
  Verdict: **NEEDS-REVISION** — algorithm structurally sound,
  four v4 spec § 7 edits required (named).
- Resolved every underspecification: ten internally in
  `algorithm.md`, four flagged as v4 spec § 7 edits the 0032
  promote-to-core session applies.
