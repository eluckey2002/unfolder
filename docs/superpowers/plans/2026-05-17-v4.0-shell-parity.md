# v4.0 — Shell parity Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Migrate `src/app/` from imperative three.js + plain DOM to React 19 + react-three-fiber + Zustand store, preserving byte-identical pipeline output and visual layout. No user-visible change.

**Architecture:** The pure `src/core/` pipeline is untouched. `src/app/` becomes a React 19 app, with the 3D viewport rendered via react-three-fiber (declarative wrapper around three.js) and the 2D pattern pane rendered as a React component that wraps the existing `emitSvg` HTML string output via `dangerouslySetInnerHTML`. A Zustand store holds the v4.0 in-memory-only state shape (sourceMesh, pipelineConfig, pinnedRegions, appliedFixes, currentLayout) — only `sourceMesh` and `currentLayout` are populated in v4.0; the rest are scaffolded for v4.1+.

**Tech Stack:** React 19, react-three-fiber 9, three.js (already vendored), Zustand 5, Immer 10, Vite (already configured), Vitest + React Testing Library + jsdom, Playwright (new — cross-browser smoke).

**Spec reference:** `docs/superpowers/specs/2026-05-16-v4-interactive-editor-design.md` §3 (v4.0 line-items), §4 (state shape), §12 (testing strategy), §9 (browser support — Chrome/Firefox/Edge primary; Safari best-effort until verified).

**Sessions:** v4.0 is sized at ~2–3 sessions per the spec. Tasks 1–4 are session 0031 (foundation: dependencies, store, configs). Tasks 5–8 are session 0032 (components + integration). Tasks 9–10 are session 0033 (replace entry + cross-browser smoke). Session numbering provisional; the implementer commits at task boundaries within sessions per existing project convention.

```tsx
import { useMemo } from "react";
import { Canvas } from "@react-three/fiber";
import { OrbitControls } from "@react-three/drei";
import { BufferAttribute, BufferGeometry } from "three";
import type { Mesh3D } from "../../core/mesh.js";

type Props = { mesh: Mesh3D | null };

const buildGeometry = (mesh: Mesh3D): BufferGeometry => {
  const positions = new Float32Array(mesh.vertices.length * 3);
  for (let i = 0; i < mesh.vertices.length; i++) {
    const [x, y, z] = mesh.vertices[i];
    positions[i * 3] = x;
    positions[i * 3 + 1] = y;
    positions[i * 3 + 2] = z;
  }
  const indices = new Uint32Array(mesh.faces.length * 3);
  for (let i = 0; i < mesh.faces.length; i++) {
    const [a, b, c] = mesh.faces[i];
    indices[i * 3] = a;
    indices[i * 3 + 1] = b;
    indices[i * 3 + 2] = c;
  }
  const geometry = new BufferGeometry();
  geometry.setAttribute("position", new BufferAttribute(positions, 3));
  geometry.setIndex(new BufferAttribute(indices, 1));
  geometry.computeVertexNormals();
  return geometry;
};

export const Viewport3D = ({ mesh }: Props): JSX.Element => {
  const geometry = useMemo(() => (mesh ? buildGeometry(mesh) : null), [mesh]);

  if (!geometry) {
    return <div className="viewport-empty">No mesh loaded.</div>;
  }

  return (
    <Canvas camera={{ position: [3, 2, 3], fov: 45, near: 0.01, far: 100 }}>
      <ambientLight intensity={0.4} />
      <directionalLight position={[5, 5, 5]} intensity={0.9} />
      <mesh geometry={geometry}>
        <meshStandardMaterial color={0x6b88a8} flatShading />
      </mesh>
      <OrbitControls enableDamping />
    </Canvas>
  );
};
```

```ts
  const { recut, pages } = runPipeline(mesh, undefined, materials);

    pages,
    pieceCount: recut.pieces.length,
  };
  set({ sourceMesh: mesh, currentLayout });
};
```
